## 🎯 Problem Statement

Every retail business faces one critical question daily:

> *"How much of each product will we sell next month — and will we have enough stock to meet that demand?"*

Getting this wrong costs money in both directions — **overstock** wastes capital and storage, **understock** loses revenue and customers. This project builds an end-to-end intelligent demand forecasting system that predicts future product sales, detects unusual sales events automatically, segments products by demand behaviour, and presents everything through a live, interactive dashboard that a business manager can use without opening Python.

## 📓 Notebook Walkthrough

### Task 1 — Data Loading, Merging & Deep Exploration
- Parses `Order Date` and `Ship Date` as proper datetime objects
- Engineers: `Year`, `Month`, `Week`, `DayOfWeek`, `Quarter`, `Season`, `ShipDays`
- Merges with supplementary dataset (multi-source practice)
- Aggregates to weekly and monthly granularity
- Answers 4 key business questions with data

### Task 2 — Time Series Analysis & Decomposition
- Plots the full 48-month sales trend
- Additive decomposition → Trend + Seasonal + Residual components
- **ADF stationarity test** (p = 0.0003 → stationary confirmed)
- First-order differencing applied and re-tested

### Task 3 — Sales Forecasting (3 Models)
Three fundamentally different approaches, trained on the same data and evaluated on the same 3-month holdout:

| Model | Approach | Key Insight |
|-------|----------|-------------|
| **SARIMA(1,1,1)(1,1,1,12)** | Classical statistical | Parameters chosen using ADF results + ACF/PACF patterns |
| **Facebook Prophet** | Decomposable trend model | Yearly seasonality dominant; weekly disabled for monthly forecasting |
| **XGBoost** | ML with lag features | Lag-1/2/3 + 3-month rolling mean + month/quarter/season as features; recursive multi-step forecast |

### Task 4 — Segment-Level Forecasting
XGBoost re-applied independently for:
- Furniture · Technology · Office Supplies
- West · East

**Strongest YoY growth forecast: East region (+25.7%)**

### Task 5 — Anomaly Detection
- **Isolation Forest** (contamination=6%) — flags 13 anomalous weeks using multi-feature scoring
- **Z-score** (±2σ from 6-week shifted rolling baseline) — flags 27 weeks
- Both methods agree on 4 weeks → **highest-priority alerts**
- Key insight: the baseline window must `shift(1)` to exclude the current point, otherwise |z| is mathematically capped at ~1.5 for a 4-point window

### Task 6 — Product Demand Segmentation
- Features: total volume, YoY growth rate, monthly sales volatility, average order value
- **Elbow method** → optimal k = 4
- **K-Means** clustering → 4 demand archetypes
- **PCA** to 2D for visualisation

| Cluster | Sub-categories | Stocking Strategy |
|---------|---------------|-------------------|
| 📦 High Volume, Stable Demand | Phones, Chairs, Tables, Storage, Binders, Accessories | Standard reorder points; safe for automation |
| 🚀 Growing Demand | Copiers | Increase safety stock; review thresholds quarterly |
| 📉 Declining Demand | Machines | Avoid forward purchase; clear excess via promotion |
| ⚡ Low Volume, High Volatility | Bookcases, Envelopes, Supplies, Art, Fasteners, Labels, Paper, Appliances, Furnishings | Smaller, more frequent orders; consider vendor-managed inventory |

## 📝 Known Limitations

1. **Small holdout set** — Model comparison uses only 3 months of holdout data. A single unusual month can shift MAPE by several points. In production, re-evaluate monthly using a rolling window.
2. **XGBoost recursive forecasting** — Each forecast step feeds its own prediction back in as input (lag features). Errors can compound over longer horizons; Month 3 forecasts should be treated with wider uncertainty than Month 1.
3. **Supplementary dataset** — `gregorut/videogamesales` is unreachable from sandboxed environments. The notebook uses a synthetic stand-in of identical schema; replace with the real Kaggle download for full compliance.
4. **No external regressors** — Models don't account for promotions, holidays, or external economic signals. A production system should incorporate these for better accuracy.
